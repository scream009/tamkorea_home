# 배포 방식 비교: Vercel vs Gabia 호스팅

사용자님의 상황(React 프로젝트, 가비아 도메인 보유)에서 어떤 방식이 더 유리한지 분석해 드립니다.

## 요약
**"Vercel 사용을 강력 추천합니다."** (이미 가비아 호스팅 결제를 하신 게 아니라면)

---

## 1. 비교표

| 비교 항목 | **Vercel (추천)** | **Gabia 웹호스팅 (FTP)** |
| :--- | :--- | :--- |
| **배포 편의성** | **완전 자동**<br>GitHub에 코드만 올리면 알아서 배포됨 | **수동**<br>코드 수정 시마다 `npm run build` -> FTP 접속 -> 파일 업로드 반복 |
| **속도 (성능)** | **매우 빠름 (Global CDN)**<br>전 세계 어디서든 빠르게 접속됨 | **보통**<br>한국 서버 기준이므로 해외에선 느릴 수 있음 |
| **비용** | **무료** (개인/소규모 기준) | **유료** (매월 호스팅 비용 발생) |
| **HTTPS (보안)** | **무료 / 자동 적용** | 별도 설정 필요하거나 유료 인증서 구매 필요 |
| **도메인 연결** | 가비아 설정 페이지에서<br>네임서버(DNS) 변경 필요 (1회) | 호스팅 구매 시 자동 연결됨 |

---

## 2. 상세 분석

### 왜 Vercel이 더 좋은가요?
1. **React/Vite 최적화**: React 앱을 위한 설정(SPA 라우팅 등)이 기본으로 되어 있어, `.htaccess` 같은 설정 파일을 만들 필요가 없습니다.
2. **자동화**: 코드를 수정하고 저장(Push)만 하면, 1분 뒤에 실제 사이트가 바뀝니다. FTP를 열 필요가 없습니다.
3. **무료 SSL**: 요즘 필수인 `https://` 보안 접속이 평생 무료입니다.

### Gabia 호스팅은 언제 쓰나요?
1. 이미 호스팅 비용(1년 치 등)을 결제했고 환불이 불가능할 때.
2. "도메인 네임서버 설정" 같은 작업이 너무 어렵고, 그냥 전화로 물어볼 수 있는 한국 고객센터가 필요할 때.

## 3. 결론 및 제안
**도메인(`tamkorea.com`)은 가비아에서 샀더라도, 호스팅(서버)은 Vercel을 쓰는 것이 "국룰(Best Practice)"입니다.**

- **제안**: 가비아 호스팅을 아직 결제하지 않으셨다면 **Vercel**로 진행하세요.
- 만약 이미 결제하셨다면, 일단 [Gabia 가이드(`GABIA_DEPLOY_GUIDE.md`)]대로 올리셔도 서비스에는 문제없습니다.
